Why it wins

A loader that acts like product infrastructure, not decoration.

The point is not to generate random bars. The point is to preserve the rhythm of the interface while the real component is still loading.

01

Inference from the React tree

Use `infer` when you want instant adoption. It walks JSX, not the browser DOM, so the runtime stays predictable.

02

Manual schema for exact shape control

Cards, chat rows, settings panels, and editorial layouts can all get placeholders that actually resemble their internals.

03

HOC wrappers for team-scale reuse

Preconfigure loading once, then expose a single `loading` prop to the rest of the application.

Exact mode

When fidelity matters, describe the internal anatomy.

Manual schema is where instaskeleton gets serious. Use `circle`, `line`, `rect`, and `group` to construct a placeholder that follows the same visual grammar as the actual component. 

const profileSchema = [
  {
    type: 'group',
    children: [
      { type: 'circle', width: '3rem', height: '3rem' },
      { type: 'line', width: '48%' },
      { type: 'line', width: '26%' }
    ]
  }
];
